# COPT / 数值求解器融合（入口说明）

这页说明如何把 **COPT / 数值求解器能力** 融合进 NSGABlack，并与启发式算法形成互补。

---

## 1) 融合目标

- 在工程框架内统一“启发式 + 精确求解”能力  
- 支持 Warm-start / 解池 / 回调 / IIS 等机制  
- 支持多策略编排与多层求解器嵌套  

---

## 2) 推荐分层位置

**Solver 层**
- 生命周期管理、评估入口、控制平面  
- 控制器仲裁、预算治理、可回放

**Adapter 层**
- 多策略组合与编排（NSGA2 / MOEA/D / VNS / SA / DE 等）  
- 支持 “外层启发式 + 内层精确求解” 的策略结构

**Evaluation（L4）**
- 把 COPT 作为评估 provider（精确/近似）  
- 统一输出目标/约束的 shape 与语义  

---

## 3) 多策略编排

常见模式：

- **串行编排**：粗搜索 → 精修  
- **并行编排**：多策略同时探索 → 最优候选融合  
- **事件驱动编排**：触发条件后切换策略  

入口参考：
- `adapter/config.py` 的 `serial / multi / event` 编排语言  

---

## 4) 多层求解器嵌套

常见结构：

```
外层（启发式/多目标） 
  └── 内层（数值求解器：COPT/LP/MIP）
```

说明：
- 外层负责策略与组合  
- 内层负责精确求解与验证  
- 内层可作为 L4 评估 provider 或内层子求解器  

---

## 5) 关键能力接口（框架级抽象）

建议把以下能力抽象为框架级接口，而非“COPT 专属”：

- **回调（Callback）**：运行过程事件点  
- **Warm-start / MIP start**：初始解引导  
- **解池（Solution Pool）**：候选解集合管理  
- **IIS（不可行子系统）**：可行性诊断  
- **参数调优**：统一调参入口与日志记录  

---

## 6) 推荐落地点

1. **L4 Provider**：COPT 作为评估提供者  
2. **Plugin**：统一回调、解池与日志  
3. **Adapter**：多策略编排与嵌套求解  

---

## 7) 适用场景

适合：
- 调度/生产计划/资源配置  
- 多目标 + 复杂约束  
- 需要工程化回放与审计的场景  

不适合：
- 单一精确求解即可覆盖的轻量问题
