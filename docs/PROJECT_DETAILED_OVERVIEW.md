## 1. 项目定位与适用问题

**NSGABlack** 是一个“偏置驱动的多目标优化生态框架”，面向以下类型的问题：

- **多目标**：多个目标同时优化（Pareto 前沿）。
- **强约束**：约束条件复杂、难以显式嵌入求解器。
- **昂贵评估**：一次评估成本高（仿真、复杂业务流程、嵌套模型等）。
- **混合变量**：连续 + 整数 + 排列 + 矩阵 + 图结构的组合。
- **黑箱问题**：目标与约束不可解析或不可导。

典型场景包括：调度、供应链、路径规划、结构设计、复杂系统参数优化等。

---

## 2. 核心理念与设计目标

本项目的目标不是“实现一版 NSGA-II”，而是构建可复用、可扩展的优化生态：

1. **解耦**

   - 算法策略与业务约束分离
   - 编码/修复/初始化/变异从求解器中剥离
2. **偏置化**

   - 强算法、启发式、规则都可以“偏置化”挂接
   - 偏置成为引导搜索的统一入口
3. **表征先行**

   - 在正确空间内初始化/变异，避免“随机 + 过滤”的低效
4. **多智能体协作**

   - 不同角色负责不同探索/开发目标
   - 通过角色比例调度实现资源分配
5. **代理加速**

   - 评估成本高时，用代理训练/筛选/替代评估
   - 代理行为本身可被偏置控制

---

## 3. 总体架构（分层视图）

```
问题层 -> 表征层 -> 偏置层 -> 求解层 -> 评估层 -> 档案/日志
                      |-> 代理层 (训练/筛选/替代评估)
                      |-> 多智能体层 (角色分工/协作/调度)
```

**层间关系要点：**

- 表征层定义“解的形式”，偏置层定义“搜索的偏好”。
- 求解层为通用 NSGA-II 或其他求解器（MOEA/D、MC、VNS 等）。
- 代理层与多智能体层是可插拔能力，可单独启用/关闭。

## 4. 核心数据流（从输入到结果）

### 4.1 单机 NSGA-II 数据流

1. 定义问题：`BlackBoxProblem`（目标 + 约束）
2. 选择表征：连续/整数/排列/矩阵/图
3. 初始化种群
4. 评估目标与约束
5. 应用偏置（奖励/惩罚/约束违规度）
6. 选择、交叉、变异
7. 环境选择（非支配排序 + 精英策略）
8. 更新 Pareto 解集

### 4.2 多智能体数据流

1. 划分角色：Explorer / Exploiter / Waiter / Advisor / Coordinator
2. 每个角色独立搜索（不同偏置或策略）
3. 角色间信息共享（选择性共享 + 档案注入）
4. 角色评分与比例自适应
5. 汇总全局 Pareto 与档案

### 4.3 代理辅助数据流

1. 真实评估采样（建模起点）
2. 代理训练（GP / RF / RBF / 集成）
3. 预筛选 / 评分偏置 / 替代评估
4. 定期重训练与质量检测

## 5. 模块详解与代码映射

### 5.1 `core/` 黑箱问题与 NSGA-II 内核

- `core/base.py`

  - `BlackBoxProblem`：统一问题接口
  - `evaluate()` / `evaluate_constraints()`
- `core/solver.py`

  - `BlackBoxSolverNSGAII`：完整 NSGA-II 内核
  - 偏置注入点与约束处理
- `core/base_solver.py`

  - 求解器公共配置与运行流程
- `core/diversity.py`

  - 多样性初始化与维护
- `core/elite.py`

  - 精英保留/替换策略与历史记忆
- `core/convergence.py`

  - 收敛检测工具
- `core/problems.py`

  - 经典基准问题与测试问题

### 5.2 `utils/representation/` 表征流水线

表征模块拆分为可组合的“编码/修复/初始化/变异/交叉”单元：

- `continuous.py`：连续变量
- `integer.py`：整数变量
- `binary.py`：二进制变量
- `permutation.py`：排列编码（TSP 等）
- `matrix.py`：矩阵编码
- `graph.py`：图结构编码

关键点：
**表征先行**使得搜索始终在正确空间进行，避免随机生成后再过滤。

### 5.3 `bias/` 偏置系统（核心特色）

偏置是项目的核心设计，分三类：

1. **算法偏置**

   - 搜索策略引导（多样性/探索/收敛等）
   - 将强算法抽象为可插拔偏置
2. **领域偏置**

   - 业务规则/约束/偏好
   - 支持硬约束与软惩罚
3. **代理控制偏置**

   - 阶段切换、预算调整、不确定性引导

关键模块：

- `bias/core/`：`BiasBase`、`AlgorithmicBias`、`DomainBias`、`OptimizationContext`
- `bias/algorithmic/`：多样性、收敛、探索等偏置
- `bias/domain/`：业务规则与约束
- `bias/surrogate/`：代理控制偏置（`PhaseScheduleBias`、`UncertaintyBudgetBias`）
- `bias/bias_compatibility.py`：兼容旧版本偏置接口
- `bias/template_*.py`：偏置模板（算法/业务/代理/函数）

### 5.4 `solvers/` 求解器入口

该目录提供多种可复用求解器：

- `nsga2.py`：NSGA-II 标准实现
- `surrogate.py`：代理辅助 NSGA-II
- `surrogate_interface.py`：统一“预筛选/评分偏置/替代评估”接口
- `moead.py`：MOEA/D
- `monte_carlo.py`：蒙特卡洛搜索
- `vns.py`：变邻域搜索
- `bayesian_optimizer.py` / `hybrid_bo.py`：贝叶斯优化相关
- `multi_agent.py`：多智能体求解器入口

### 5.5 `multi_agent/` 多智能体系统

多智能体层强调“角色分工 + 策略组合 + 资源分配”：

- **核心角色**

  - Explorer：探索新区域
  - Exploiter：强化局部最优
  - Waiter：备用池，可动态转岗
  - Advisor：候选生成与建议（可接代理偏置/贝叶斯偏置）
  - Coordinator：统筹角色比例与区域划分
- **核心模块**

  - `multi_agent/core/`：角色、种群、基础结构
  - `multi_agent/components/`：通信、进化、档案、多层评分
  - `multi_agent/bias/`：内置偏置与角色配置
  - `multi_agent/strategies/`：搜索策略模块
- **关键机制**

  - 多层档案：可行档案/边界档案/多样性档案
  - 角色评分偏置：偏置可作为评分器
  - 角色比例自适应：按表现动态调整
  - 选择性共享：不同角色获得不同类型的共享信息

### 5.6 `surrogate/` 代理模型系统

代理系统提供从训练到调用的完整流程：

- `surrogate/base.py`：基础代理接口
- `surrogate/trainer.py`：训练与复训练
- `surrogate/evaluators.py`：评估与筛选
- `surrogate/strategies.py`：采样与更新策略
- `surrogate/manager.py`：代理生命周期管理
- `surrogate/features.py`：特征处理
- `surrogate/utils.py`：工具函数

代理支持：GP / RF / RBF / 集成模型，并包含质量检测与持久化。

### 5.7 `tools/` 工程与文档工具

- `tools/build_catalog.py`：生成项目目录索引
- `tools/generate_api_docs.py`：自动生成 API 文档

### 5.8 `examples/` 示例与验证

示例覆盖：TSP、矩阵表示、代理优化、多智能体、偏置流程等。
建议优先运行 `examples/validation_smoke_suite.py` 进行全局冒烟验证。

## 6. 设计小巧思与创新点（扩展版）

以下“巧思”按 **结构层 / 策略层 / 代理层 / 工程层** 分组，便于对外讲清楚“创新在哪里”。

### 6.1 结构巧思（架构层）

1. **约束外置**

   - 约束不侵入求解器内核，通过偏置返回约束违规度，统一进入排序与筛选
2. **偏置统一返回协议**

   - 偏置可返回 `penalty / reward / constraint` 等字段，兼容“惩罚式”和“评分式”思路
3. **偏置兼容层**

   - 老接口与新接口并存，减少历史代码破坏，降低迁移成本
4. **偏置可叠加组合**

   - 多个偏置可以叠加使用，形成“策略组合”，不需要重写求解器
5. **表征先行**

   - 在合法空间内直接生成/变异，避免“随机 + 过滤”的效率损失
6. **算子模块化**

   - 交叉/变异/修复拆分为独立模块，可替换、可复用、可按问题快速组合
7. **偏置与表征解耦**

   - 同一偏置可应用于不同表征，降低策略与变量类型耦合度
8. **稳定底座 + 可插拔上层**

   - NSGA-II 作为稳态底座，偏置/代理/多智能体作为可插拔扩展层
9. **偏置集中管理**

   - 通过注册/管理器统一挂载偏置，避免散落到求解流程中
10. **统一优化上下文**

- `OptimizationContext` 统一提供代数、指标、历史与问题信息，偏置与 Advisor 共享语义

11. **表征管线协议化**

- `RepresentationPipeline` 用 Protocol 约束编码/修复/初始化/变异接口，避免强耦合

12. **单机与多智能体同构接口**

- 表征与偏置在单机/多智能体共享，迁移到新问题无需重写核心流程

### 6.2 策略巧思（搜索层）

1. **三层档案机制**

   - 可行档案 + 边界档案 + 多样性档案并行维护，兼顾质量与探索
2. **选择性共享**

   - Explorer 优先拿多样性档案，Exploiter 优先拿最优档案，策略与角色匹配
3. **角色评分偏置**

   - 偏置可作为评分器注入，直接影响角色比例与资源分配
4. **角色比例自适应**

   - 按改进率、可行率、多样性综合评分，动态调节各角色规模
5. **Advisor 候选多源**

   - 候选可来自档案 + 随机采样 + 代理/贝叶斯偏置，保持策略多样性
6. **Waiter 作为动态资源池**

   - 可随时转岗到 Explorer/Exploiter 等角色，实现资源弹性调度
7. **Coordinator 统筹机制**

   - 依据轮次、质量与收敛情况进行区域划分或角色重配
8. **历史记忆 + 精英替换**

   - 保留历史优秀解并进行替换，兼顾稳定性与跳出局部最优
9. **多样性感知初始化 + 历史去重**

   - 候选池 + 相似度阈值 + 历史拒绝，减少重复探索
10. **自适应变异范围**

- 变异范围随代数逐步收缩，前期探索、后期收敛

11. **收敛监测与噪声容忍**

- 停滞/多样性/噪声联合判定，降低误判与过早停止

12. **历史扰动补充精英**

- 智能历史生成扰动候选用于替换，提升跳出局部最优能力

13. **阶段性策略调节**

- 早期强化探索，后期强化开发，偏置参数随进度自动调整

### 6.3 代理巧思（加速层）

1. **代理三种入口统一**

   - 预筛选 / 评分偏置 / 替代评估采用统一接口配置
2. **代理控制偏置**

   - 阶段切换、预算调整、不确定性利用可由偏置控制
3. **质量检测与自动复训**

   - R2/RMSE 监控，质量下降自动触发复训
4. **模型持久化复用**

   - 训练好的代理可保存/加载，避免重复成本
5. **多模型集成**

   - GP/RF/RBF 集成提升稳定性，减少单模型不可靠风险
6. **多目标代理分目标训练**

   - 多目标时为每个目标训练独立模型，减少目标间干扰
7. **真实评估预算调度**

   - 真实评估与代理评估比例可控，适合昂贵评估场景
8. **不确定性驱动采样**

   - GP 不确定性用于挑选真实评估点，提高样本价值

### 6.4 工程巧思（落地层）

1. **并行评估接口**

   - 支持批量评估与并行评估，提高昂贵函数吞吐
2. **冒烟验证脚本**

   - `examples/validation_smoke_suite.py` 一键验证核心模块
3. **文档/目录索引自动化**

   - `tools/build_catalog.py` / `tools/generate_api_docs.py` 自动整理项目结构
4. **对外入口精简**

   - 统一导入入口，降低对外展示的复杂度
5. **示例驱动**

   - 大量示例覆盖核心能力，降低新用户理解成本
6. **就地说明文档**

   - 关键类旁边直接放置说明文档，降低“忘记自己做过什么”的成本
7. **模板化扩展**

   - 偏置模板与快速入口明确，降低新增策略的心智负担
8. **快速非支配排序加速**

   - `fast_non_dominated_sort` + Numba 加速，失败自动回退
9. **并行评估稳健性**

   - 负载均衡 + 重试机制 + 串行回退，避免单点失败
10. **内存优化器**

- 自动清理与历史压缩，长时间运行更稳定

11. **实验结果容器**

- `ExperimentResult` 保存 Pareto / 配置 / 历史，便于复现

12. **交互式可视化**

- Matplotlib 面板直接调参、开关策略、观察前沿变化

13. **双路径导入容错**

- 包模式/脚本模式均可运行，降低初次使用门槛

### 6.5 协作细节巧思（多智能体）

1. **档案注入带抖动**

   - 档案候选注入时加入轻微扰动，避免种群塌缩
2. **区域划分与缩放**

   - 根据高质量解划分区域边界，按角色倍率缩放搜索范围
3. **Waiter 动态转岗**

   - 备用池按轮次与策略被调配到高潜力角色
4. **Advisor 候选源权重**

   - 候选来源可配置权重与池大小，兼顾档案/随机/模型建议
5. **全局最优传播但保留独立性**

   - 全局最优共享到部分角色，同时保留关键角色的独立探索

## 7. 扩展与二次开发指南（框架级）

### 7.1 新增偏置

根据类型选择模板：

- 算法偏置：`bias/algorithmic/template_algorithmic_bias.py`
- 业务偏置：`bias/domain/template_domain_bias.py`
- 代理偏置：`bias/surrogate/template_surrogate_bias.py`
- 函数偏置：`bias/template_function_bias.py`

编写后注册到偏置管理器即可使用。

### 7.2 新增表征类型

步骤：

1. 在 `utils/representation/` 下新增模块
2. 实现初始化、修复、变异、交叉接口
3. 在管线中组合/调用

### 7.3 新增多智能体角色

步骤：

1. 角色枚举添加新角色
2. 角色配置/偏置配置加入
3. 决策与共享逻辑可按需扩展

---

## 8. 新问题落地流程

1. **定义问题接口**

   - 目标 + 约束实现于 `BlackBoxProblem`
2. **确定表征管线**

   - 连续/整数/排列/矩阵/图
3. **选择偏置组合**

   - 算法偏置（策略）
   - 领域偏置（规则/约束）
   - 代理偏置（预算/阶段/不确定性）
4. **选择求解器**

   - 单机 NSGA-II
   - 代理辅助 NSGA-II
   - 多智能体协作
5. **运行与验证**

   - 运行示例或 `validation_smoke_suite`
   - 观察日志与 Pareto 结果

## 9. 对外展示价值总结

NSGABlack 的展示价值在于：

- 提供完整的“问题 → 表征 → 偏置 → 求解 → 代理/多智能体”链路
- 体现“模块化 + 偏置化 + 可插拔”的工程能力
- 支持从教学演示到复杂应用的可扩展路径
- 有明确的扩展机制与代码层次结构，适合展示系统化能力
